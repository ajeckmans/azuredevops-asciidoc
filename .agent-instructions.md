# Agent Instructions for Azure DevOps AsciiDoc Extension

When working on this repository, please adhere to the following guidelines:

## Core Purpose
This extension renders `.adoc` / `.asciidoc` files in a custom Pull Request tab inside Azure DevOps. It also supports file-level commenting directly from the rendered AsciiDoc view.

## Tech Stack
- **Framework:** React + TypeScript
- **UI:** `azure-devops-ui`
- **SDK & API:** `azure-devops-extension-sdk`, `azure-devops-extension-api`
- **AsciiDoc Engine:** `@asciidoctor/core`
- **Bundler:** Webpack

## Important Design Decisions
- **Comments:** Comments are currently implemented at the **file-level** using `GitRestClient.createThread`. We do not have line-level sourcemap bindings yet.
- **Project Structure:**
  - `vss-extension.json`: The manifest (Publisher: `ajeckmans`, ID: `asciidoc`).
  - `src/App.tsx`: Main entry, manages state and initialization.
  - `src/services/DevOpsService.ts`: Centralizes all calls to Azure DevOps API.
  - `src/components/FileTree.tsx`: Displays the `.asciidoc` files found in the source branch.
  - `src/components/AsciiDocRenderer.tsx`: Converts AsciiDoc content to HTML and displays the comment input form.

## Ways of Working (WoW)
- Please check `WoW.md` for standard practices.
- Ensure all new features align with the inspiration taken from `format-check-task`.
- Run `npm run build` to verify any changes.
